Routes: chat.js

      GET -> /thread/                     -> used to display all the thread info at left side based on updated_At time
      GET -> /thread/:threadId            -> to display specific thread info. when we click on thread on left side (history) to display that thread info we use it
      DELETE -> /thread/:threadId         -> to delete a specific thread
    * POST -> /chat                       -> to display new messages and reply from backend

In POST -> /chat

      - we already a /api route for gpt api call

      when we type message in message box in frontend , after clicking submit button it goes to backend with route "/api/chat"
      and in that /api/chat it consist of threadId and content.

      and backend call the api call for openRouter api and it sends reply to the backend. and backend send response to the frontend.

      after reply send to frontend we must save that reply message into db. and also change updateAt date with present date


      first validate the theadId and message

      if threadId is already exist in dababase (that means we using history of chats) then we simply push that message in content in db using threadId.

      if threadId is not exist in db (then it is a new chat) then we create a new thread in db with threadId, content


for route checking go to thunder client extension in vs code click new request

for new chat or existing chat 

      POST  http://localhost:8080/api/chat

For delete chat 

      DELETE http://localhost:8080/api/thread/<threadId>

Get specific chat

      GET http://localhost:8080/api/thread/<threadId>
